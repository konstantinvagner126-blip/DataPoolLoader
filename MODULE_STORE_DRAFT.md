# Module Store Draft

Черновик. Не является утвержденной частью roadmap/backlog.

## Идея

Ввести для UI два взаимоисключающих режима работы с модулями:

- `files`
- `database`

Гибридный режим не использовать.

## Зачем

- сохранить текущий файловый сценарий для тех, у кого нет доступа к БД;
- подготовить переход к хранению модулей в БД без немедленного отказа от `apps`;
- дать UI возможность работать с большим количеством модулей через централизованный реестр.

## Режим `files`

В этом режиме:

- UI работает только с каталогом `apps`;
- список модулей берется из файлов;
- редактирование идет по файлам;
- запуск идет по файлам;
- PostgreSQL для реестра модулей не требуется.

## Режим `database`

В этом режиме:

- UI работает только с PostgreSQL-реестром модулей;
- список модулей берется из БД;
- редактирование должно идти через БД;
- запуск должен идти из данных, хранящихся в БД;
- `appsRoot` нужен как источник для загрузки модулей в БД.

## Операции синхронизации

Для режима `database` должны быть предусмотрены явные операции:

- `Обновить все модули в БД`
- `Обновить выбранный модуль в БД`

Предварительное направление:

- источник импорта на переходном этапе — `appsRoot`;
- UI должен уметь переносить metadata, конфиг и SQL-ресурсы модуля в PostgreSQL;
- удаление `apps` откладывается на самый конец миграции.

## Ограничения, зафиксированные на сейчас

- PostgreSQL для режима `database` обязателен;
- режимов только два: `files` и `database`;
- без гибридного режима;
- пока не удаляем `apps`;
- сначала прорабатываем детали и только потом вносим это в `ROADMAP.md` и `BACKLOG.md`.

## Открытые вопросы для дальнейшей проработки

- точная схема таблиц PostgreSQL для модулей и SQL-ресурсов;
- как хранить versioning/updated-at/hash модулей;
- как обрабатывать удаленные из `apps` модули при полной синхронизации;
- как будет устроено создание нового модуля через UI;
- когда именно запуск в режиме `database` окончательно перестанет зависеть от файлов;
- какой конфиг UI будет управлять переключением режима и параметрами подключения к PostgreSQL.

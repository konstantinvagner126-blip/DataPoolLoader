Раздел 1

Модуль загрузки данных

Используется для выгрузки данных из нескольких PostgreSQL-источников, сборки `merged.csv` и загрузки результата в target-таблицу.
Открыть модуль
Что делает
Export -> Merge -> Import
По каждому source создается отдельный CSV, затем формируется общий файл и при необходимости выполняется загрузка в target.
Где настраивается
application.yml
В каждом app-модуле используется собственный `src/main/resources/application.yml` и связанные SQL-файлы.
Как запускать
UI или Gradle
Можно запускать из UI, из IDEA или через `./gradlew :apps:<module>:run`.
Что должно быть в конфиге
Общие параметры: `outputDir`, `mergeMode`, `parallelism`, `fetchSize`, `queryTimeoutSec`.
SQL: либо `commonSql`/`commonSqlFile`, либо `sources[].sql`/`sources[].sqlFile`.
Источники: `sources[].name`, `jdbcUrl`, `username`, `password`.
Квоты: `quotas` используются только при `mergeMode=quota`.
Target: `enabled`, `jdbcUrl`, `username`, `password`, `table`, `truncateBeforeLoad`.
Значения по умолчанию
outputDir`./output`
mergeMode`plain`
parallelism`5`
fetchSize`1000`
progressLogEveryRows`10000`
target.enabled`true`
Пример application.yml
app:
  outputDir: ./output
  mergeMode: round_robin
  parallelism: 5
  fetchSize: 1000
  queryTimeoutSec: 60
  commonSqlFile: classpath:sql/common.sql

  sources:
    - name: db1
      jdbcUrl: ${DB1_JDBC_URL}
      username: ${DB1_USERNAME}
      password: ${DB1_PASSWORD}

    - name: db2
      jdbcUrl: ${DB2_JDBC_URL}
      username: ${DB2_USERNAME}
      password: ${DB2_PASSWORD}
      sqlFile: classpath:sql/db2_override.sql

  target:
    enabled: true
    jdbcUrl: ${TARGET_JDBC_URL}
    username: ${TARGET_USERNAME}
    password: ${TARGET_DB_PASSWORD}
    table: public.test_data_pool
    truncateBeforeLoad: true
Как работает credential.properties
Назначение: подставляет значения для `${...}` в `jdbcUrl`, `username`, `password`.
Поиск: `-Dcredentials.file` -> `gradle/credential.properties` -> `~/.gradle/credential.properties`.
В UI: можно загрузить файл через интерфейс, он имеет приоритет над fallback-поиском.
DB1_JDBC_URL=jdbc:postgresql://localhost:5432/db1
DB1_USERNAME=user1
DB1_PASSWORD=secret1

TARGET_JDBC_URL=jdbc:postgresql://localhost:5432/target_db
TARGET_USERNAME=loader
TARGET_DB_PASSWORD=target_secret
Что важно помнить
  • Если `target.enabled=false`, выгрузка и merge выполняются, но импорт в target пропускается.
  • `commonSql` и `commonSqlFile` нельзя задавать одновременно. То же правило действует для `source.sql` и `source.sqlFile`.
  • Для `classpath:sql/...` путь считается относительно `src/main/resources`.
  • В UI можно менять конфиг через визуальную форму и `application.yml`; изменения синхронизируются между собой.
Раздел 2

SQL-консоль

Инструмент для ручной работы с настроенными источниками: проверки `SELECT`, служебных DDL/DML-команд и диагностики данных.
Открыть SQL-консоль
Источники
ui.sqlConsole.sources
SQL-консоль не использует app-модули. Она читает собственный список источников из UI-конфига.
Режим выполнения
1 statement
За один запуск выполняется один SQL statement. Для `SELECT` данные показываются отдельно по каждому source.
Управление
Async + Cancel
Запрос выполняется асинхронно, в UI виден статус и есть кнопка отмены длительной операции.
Как настроить источники
ui:
  sqlConsole:
    fetchSize: 1000
    maxRowsPerShard: 200
    queryTimeoutSec: 60
    sources:
      - name: db1
        jdbcUrl: ${DB1_JDBC_URL}
        username: ${DB1_USERNAME}
        password: ${DB1_PASSWORD}
      - name: db2
        jdbcUrl: ${DB2_JDBC_URL}
        username: ${DB2_USERNAME}
        password: ${DB2_PASSWORD}
Как пользоваться
  1. Открой SQL-консоль с главной страницы.
  2. Проверь, что `credential.properties` загружен, если в `ui.sqlConsole.sources` используются `${...}`.
  3. Отметь чекбоксами нужные sources.
  4. Введи SQL и нажми `Выполнить`.
  5. Если запрос работает долго, следи за статусом и при необходимости нажми `Отменить`.
Как выглядит результат
Для SELECT: отдельные вкладки/результаты по каждому source, без смешивания строк между источниками.
Для DDL/DML: сводка по каждому source со статусом, `affected rows` и текстом ошибки, если она была.
Ограничение выдачи: используется `maxRowsPerShard` из UI-конфига.
Что важно помнить
  • SQL-консоль использует свой timeout: `ui.sqlConsole.queryTimeoutSec`.
  • Если в `ui.sqlConsole.sources` используются placeholders, без credentials часть подключений не разрешится.
  • Сейчас поддерживается один SQL statement за запуск; `multi-statement SQL` пока отложен.
